# MRO 구매 검토 시스템 — PRD (Product Requirements Document)

> 작성일: 2026-03-31
> 버전: 1.0
> 스택: Next.js 16.2.1 (App Router) · React 19 · Recharts 3.8.0 · xlsx 0.18.5 · Tailwind CSS v4 · TypeScript 5

---

## 1. 개요

### 1.1 목적

제조부문 MRO(Maintenance, Repair & Operations) 구매 시, 2024~2025년 과거 구매 이력을 기반으로 2026년 이후 구매 건의 **적절성을 정량적으로 검토**할 수 있는 내부 웹 도구.

### 1.2 배경

- MRO 구매는 건별 금액이 크지 않아 개별 검토가 누락되기 쉬움
- 과거 단가·수량·주기 데이터가 있어도 담당자가 직접 확인하기 어려움
- 이상 징후(단가 급등, 조기 구매, 수량 급증)를 자동 감지하여 검토 부담 경감

### 1.3 사용자

- 구매 담당자 (구매 건 검토 시 사용)
- 구매 관리자 (이상 징후 모니터링)

## 2. 기능 요구사항

### 2.1 데이터 관리

#### 기준 데이터 (읽기 전용)
| 파일 | 기간 | 비고 |
|------|------|------|
| `결재이력(상품단위) 2024년 전체.xls` | 2024년 전체 | 변경/삭제 불가 |
| `결재이력(상품단위) 2025년 전체.xls` | 2025년 전체 | 변경/삭제 불가 |

#### 2026년 월별 파일 업로드
- 웹 UI에서 `.xls` / `.xlsx` 파일 업로드 가능
- 업로드된 2026년 파일은 삭제 가능 (기준 데이터는 삭제 불가)
- 업로드 후 데이터 캐시 자동 무효화 (30초 인메모리 캐시)

#### Excel 파일 구조
```
Row 0: 타이틀 ("결재이력(상품단위)")
Row 1: 컬럼 헤더 (주문번호, 상품명, 규격, 제조원명, 사업장/부서명, 주문자, 주문대행자, 주문일, 총수량, 총금액, 통화단위, 결재일자, 결재처리, 구매사유, 수령인)
Row 2+: 데이터
```

---

### 2.2 상품 검색

#### 검색창
- 2글자 이상 입력 시 **실시간 자동완성** (200ms 디바운스)
- 자동완성 결과:
  1. 완전 일치 (정규화 후)
  2. 포함(contains) 일치 — 입력어가 상품명에 들어있는 모든 항목
  3. Levenshtein 유사도 0.75 이상 — 오타·유사명 대응
- **띄어쓰기 정규화**: 연속 공백 → 단일 공백으로 자동 처리 (예: "포스트  잇" = "포스트잇")
- 유사도 90% 이상 품목은 별도 섹션 F에서 표시

#### 규격 검색
- 상품 검색 후 해당 상품의 규격 목록이 자동 추출됨
- 규격 입력창에서 **포함(contains) 검색** (대소문자 무관)
- 규격 입력 시 300ms 디바운스 후 자동 재조회
- 규격 필터 적용 시 상단 뱃지로 표시

---

### 2.3 부서 필터

3단계 계층 구조 탭:

```
1단계 (본부): 품질본부 · 생산본부 · 운영지원본부
    ↓
2단계 (담당): 본부 내 담당 조직
    ↓
3단계 (팀): 담당 내 개별 팀
```

- 각 단계 선택 시 하위 탭 자동 표시
- 필터 변경 시 현재 상품 자동 재조회
- 선택된 본부/담당/팀은 상품명 옆 뱃지로 표시
- 전체 선택 가능 (필터 없음)

---

### 2.4 발주유형 구분

- 주문번호 앞자리로 자동 분류:
  - `2*` → 일반발주 (초록색 막대)
  - `7*` → 선발주 (주황색 막대)
- **구매 시점별 수량 타임라인** 차트 내 체크박스로 표시/숨김 토글
- 두 유형 동시 표시 시 **스택 막대 그래프**로 구분

---

### 2.5 분석 결과 화면 (6개 섹션)

검색 후 아래 섹션이 순서대로 표시됨.

---

#### 섹션 A — 구매 시점별 수량 타임라인

**목적**: 언제, 얼마나 샀는지 시간 흐름으로 파악. "3개월마다 사는 품목인데 지금 사는 게 이른가?" 한눈에 확인.

| 항목 | 설명 |
|------|------|
| X축 | 전체 기간의 **월별 라벨 전부 표시** (`'YY.01` / `MM` 형식) |
| Y축 | 수량 (개) |
| 막대 | 일반발주(초록) + 선발주(주황) 스택, 구매 없는 달은 빈 막대 |
| 검토 중인 건 | 오늘 달에 **빨간 반투명 막대** (체크박스 유무와 무관하게 항상 표시) |
| 현재 시점 | 빨간 점선 세로선 (`▼ 현재`) |
| 예상 다음 구매 | 초록 점선 세로선 (`▼ 예상(N일)`) — 이번 달인 경우 `◀ 예상(이번달)` |
| 범위 초과 시 | 예상 시점이 차트 밖이면 우측에 텍스트 안내 배너 |

하단 요약 카드:
- 총 구매 횟수
- 평균 구매 주기 (일)
- 마지막 구매일
- 마지막 구매 후 경과 (주기 대비 %)

---

#### 섹션 B — 연도별/월별 구매금액 추이

| 항목 | 설명 |
|------|------|
| X축 | 월별 라벨 전부 표시 |
| 왼쪽 Y축 | 구매 건수 (막대, 연도별 색상) |
| 오른쪽 Y축 | 총금액 (꺾은선) |
| 검토 중인 건 | 오늘 달에 **빨간 반투명 막대** (금액 기준) |
| 현재 시점 | 빨간 점선 세로선 |
| 예상 다음 구매 | 초록 점선 세로선 (범위 자동 확장) |
| 기준선 | 과거 월평균 금액 (회색 점선) |

---

#### 섹션 C — 단가 변동 이력

| 항목 | 설명 |
|------|------|
| X축 | 월별 라벨 전부 표시 |
| Y축 | 단가 (원) |
| 데이터 포인트 | 2024~2025: 초록 점 / 2026: 주황 점 |
| 기준선 | 과거 평균 (회색) · 150% (주황) · 200% (빨강) |
| 검토 중인 건 | 수평 빨간 점선 + 다이아몬드 마커 |
| 현재 시점 | 빨간 점선 세로선 |

---

#### 섹션 D — 구매 부서/공장 분포

- 파이 차트 (전체 기간)
- 부서별 구매 금액 비율 표시

---

#### 섹션 E — 2026년 구매 건 검토 (이상 탐지)

각 2026년 구매 건에 대해 이상 플래그 자동 생성:

| 플래그 | 조건 | 심각도 |
|--------|------|--------|
| 단가 급등 | 과거 평균 대비 200% 초과 | 🔴 위험 |
| 단가 급등 | 과거 평균 대비 150~200% | 🟡 주의 |
| 수량 급증 | 과거 월평균 대비 300% 초과 | 🟡 주의 |
| 신규 고액 품목 | 이력 없고 금액 50만원 이상 | 🟡 주의 |
| 신규 품목 | 이력 없고 금액 50만원 미만 | ℹ️ 참고 |
| 구매 빈도 증가 | 과거 연 2회 이하인데 2026년 2회 이상 | ℹ️ 참고 |

---

#### 섹션 F — 유사 상품명

- Levenshtein 유사도 90% 이상 품목 표시
- 클릭 시 해당 상품 바로 검색

### 2.6 검토 중인 구매 건 입력 및 종합 판정

#### 입력 항목
| 필드 | 설명 |
|------|------|
| 수량 | 검토 중인 구매 수량 |
| 단가 | 검토 중인 단가 |
| 총금액 | 자동 계산 (수량 × 단가) |

- 부서 필터 또는 규격 변경 시에도 **입력값 유지** (재입력 불필요)

#### 종합 판정 카드 (3개 항목 비교)

**① 단가 비교**
- 검토 단가 vs 과거 평균 단가
- 비율 게이지 바 (과거 평균 = 50% 기준선)
- 과거 최저~최고 범위 표시

| 비율 | 판정 |
|------|------|
| ≤ 110% | ✅ 적정 |
| 110~150% | 🟡 주의 |
| 150~200% | ⚠️ 경고 |
| > 200% | 🔴 위험 |

**② 수량 비교**
- 검토 수량 vs 과거 건당 평균 수량

**③ 구매 시점**
- 마지막 구매 후 경과일 vs 평균 구매 주기
- 진행 게이지 바 (80% 마커 = 권장 구매 시작 기준)

| 주기 경과 | 판정 |
|-----------|------|
| ≥ 80% | ✅ 적정 |
| 50~80% | 🟡 주의 |
| 30~50% | ⚠️ 경고 |
| < 30% | 🔴 위험 |

**종합 판정**: 세 항목 중 가장 나쁜 등급 기준, 판정 사유 텍스트 포함

> 최종 승인 판단은 담당자가 합니다. 기준값은 참고용이며 맥락에 따라 해석이 달라질 수 있습니다.

## 3. 비기능 요구사항

### 3.1 성능
- Excel 데이터 30초 인메모리 캐시 (파일 업로드 시 즉시 무효화)
- 검색 자동완성 200ms 디바운스
- 규격 필터 변경 300ms 디바운스

### 3.2 보안
- 비밀번호 기반 접근 제어 (서버 측 미들웨어)
- 세션은 브라우저 종료 시 만료 (쿠키 maxAge 없음)
- 2024~2025 기준 데이터 파일 삭제 API 보호 (HTTP 403)

### 3.3 배포
- Vercel (GitHub 연동, 자동 재배포)
- Root Directory: `MROreviewer/web`
- 데이터 파일은 `MROreviewer/web/data/`에 포함하여 빌드
