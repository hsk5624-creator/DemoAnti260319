# APQR Agent — 연간품질평가 자동 작성 보조 시스템 PRD

> 작성일: 2026-04-13
> 버전: 1.1

---

## 1. 개요

### 1.1 목적
GMP 의무 문서인 **연간품질평가(APQR)** 보고서 작성 시,
분산된 Excel·PDF 데이터 파일을 업로드하면 **정해진 Word 양식의 표와 서술 항목에 데이터를 자동으로 채워주는** 웹 도구.

AI 호출 없음. 과금 없음. 순수 데이터 파싱 + 문서 자동완성.

### 1.2 문제 상황
- APQR 작성자는 배치 기록, 시험 성적서, 일탈 현황, 안정성 결과 등 수십 개 파일을 수동으로 확인·정리해야 함
- 여러 파일에서 숫자를 찾아 Word 표에 하나씩 옮겨 붙이는 단순 반복 작업이 대부분
- 담당자 교체 시 작성 기준이 일관되지 않아 품질 편차 발생

### 1.3 목표
- Excel/PDF 파일 업로드 → 데이터 자동 추출
- 추출된 데이터를 Word 양식의 해당 위치에 자동 삽입
- 웹에서 결과 미리보기 후 Word 파일 다운로드

## 2. 사용 흐름 (User Flow)

```
① Word 양식 업로드 (.docx)
      ↓
   표 헤더·섹션 구조 파싱 → 어떤 데이터가 필요한지 파악
      ↓
② 데이터 파일 업로드 (Excel / PDF — 1개씩 반복 가능)
      ↓
   파일에서 표·텍스트 데이터 추출
      ↓
   사용자가 "이 파일 → 이 섹션" 매핑 지정
      ↓
③ [보고서 생성] 버튼 클릭
      ↓
   Word 양식의 각 표/항목에 데이터 삽입
      ↓
④ 웹 미리보기 확인
      ↓
⑤ Word 파일 다운로드
```

## 3. 기능 요구사항

### 3.1 Word 양식 업로드 및 구조 파싱

- `.docx` 파일 업로드
- 양식 내 **표(Table)** 자동 인식
  - 첫 행(헤더)을 읽어 어떤 데이터가 들어가는 표인지 식별
  - 예: `["배치번호", "제조일", "수율(%)", "판정"]` → 배치 현황 표
- 양식 내 **섹션 제목** 자동 인식
  - 굵은 글씨 또는 번호 체계로 섹션 목록 구성
- 파싱 결과: 섹션 목록 + 각 섹션의 표 헤더 표시

### 3.2 데이터 파일 업로드

#### Excel (.xlsx / .xls)
- 라이브러리: `xlsx` (SheetJS)
- 시트 목록 자동 표시 → 사용자가 사용할 시트 선택
- 헤더 행 자동 감지
- 추출 결과: `{ [열명]: 값 }[]` 형태의 행 배열

#### PDF (.pdf)
- 라이브러리: `pdf-parse` (서버 사이드 Next.js API route)
- 텍스트 전체 추출 후 줄 단위 파싱
- 표 형태 데이터는 공백/탭 패턴으로 열 분리 시도
- 추출 결과 품질이 낮으면 원문 텍스트 그대로 표시하고 사용자가 확인

#### 업로드 파일 관리
- 업로드된 파일 카드 목록으로 표시
- 카드에: 파일명 / 추출된 데이터 미리보기(상위 5행) / 매핑된 섹션
- 파일 삭제 및 재업로드 가능

### 3.3 섹션 매핑

- 사용자가 **"이 파일의 이 열 → Word 양식의 이 섹션(표)"** 을 지정
- 매핑 방식:
  - 파일의 열 헤더와 Word 표의 열 헤더가 동일하면 **자동 매핑**
  - 이름이 다르면 드롭다운으로 수동 연결
- 매핑 결과는 저장되어 다음 파일 업로드 시 재사용 가능 (localStorage)

### 3.4 보고서 데이터 삽입

- Word 양식의 표: 매핑된 데이터 행 순서대로 삽입
  - 기존 템플릿 행(헤더 아래 빈 행)을 기준으로 반복 생성
- 서술형 섹션: 현재는 공란 유지 (향후 통계 자동 계산 추가 예정)
- 삽입 시 원본 양식의 **폰트·셀 스타일 유지**

### 3.5 웹 미리보기

- 생성된 보고서를 섹션 탭으로 표시
- 표 데이터: HTML 테이블로 렌더링
- 수치 확인 후 이상 있으면 소스 파일 재업로드 또는 값 직접 수정 가능

### 3.6 Word 파일 다운로드

- 라이브러리: `docxtemplater` + `pizzip`
- 원본 양식의 레이아웃·스타일 유지한 채 데이터만 삽입
- 파일명: `APQR_[제품명]_[연도]_완성본.docx`

## 6. 데이터 모델

```typescript
// 양식에서 파싱된 섹션
interface Section {
  id: string;
  title: string;
  type: "table" | "text";
  columns?: string[];   // 표인 경우 열 헤더
}

// 업로드된 데이터 파일
interface DataFile {
  id: string;
  fileName: string;
  fileType: "excel" | "pdf";
  sheets?: SheetData[];        // Excel의 경우
  rawText?: string;            // PDF의 경우
  mappedSectionId: string | null;
}

// 시트 데이터
interface SheetData {
  name: string;
  headers: string[];
  rows: Record<string, string>[];
}

// 섹션-파일 매핑
interface ColumnMapping {
  sectionId: string;
  fileId: string;
  sheetName?: string;
  columnMap: Record<string, string>;  // { Word열명: 파일열명 }
}
```
